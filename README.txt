【工具說明】
  純前端單機網頁工具（index.html 雙擊即可開啟，無需安裝、無需伺服器）
  從 Excel 規則文件自動產生 1920×1080 像素的遊戲 Help 說明圖。
  支援匯出單頁 PNG 或全部頁面 ZIP，以及工作紀錄 JSON 儲存/載入。


================================================================================
  快速開始
================================================================================

1. 雙擊 index.html 以瀏覽器開啟
2. 【檔案匯入】區：
   - 選取 Help 規則 Excel 檔（.xlsx / .xls）
   - 選取底圖（1920×1080 PNG/JPG）
   - 選取 Logo 圖片
   - 選取小圖示資料夾（資料夾內放所有 PNG/JPG 圖示）
3. 點選左側頁面列表預覽各頁
4. 按下【偵測掉字】確認所有內容無遺漏
5. 匯出 PNG 或 ZIP


================================================================================
  Excel 文件格式規範
================================================================================

  A 欄：群組名稱（同一群組請留空，只在第一行填寫）
  B 欄：類型標籤
  C 欄：內容文字

【B 欄類型標籤說明】
  標題       → 該符號/功能的主標題（可含圖片，格式：標題文字[TOKEN]）
  頁首       → 頁面頂部說明文字（鐵則：永遠顯示在最上方，不論在標題前或後）
  頁尾       → 頁面底部法律聲明
  規則       → 規則說明文字
  賠分表     → 賠率表格資料（格式：[TOKEN] N-M，多組可同行）
  賠率表     → 同上
  表格       → 管線表格（用 ｜ 符號分隔欄位）
  球分表     → 同上

【A 欄群組類型】
  特殊標誌   → 每個子標題獨立成頁，共享頁首/頁尾
  轉輪示意   → 每頁最多 2 個區塊左右並排
  其他群組   → 一般頁面，自動分頁


================================================================================
  內容標記格式
================================================================================

  [TOKEN]      → 置入對應名稱的圖示（方括號，全形或半形皆可）
  {N}          → 數字，套用大號黃色數字樣式
  ｜           → 表格欄位分隔線（管線表格用）
  ----------   → 10個以上的短橫線 → 渲染為實心分隔線

【特殊情境】
  - Token 名稱含「示意圖」：以原始尺寸渲染（自動縮放以保留規則文字空間）
  - 3個以上連續空白隔開的兩段文字：自動識別為隱式兩欄表格
  - N-M 格式（如 5-250）：自動識別為分數表（Score Table）

【Excel 儲存格多行】
  儲存格內的換行（Alt+Enter）會自動展開為獨立行，
  支援分數表（每行一組 N-M）或規則等多行場景。


================================================================================
  版面說明
================================================================================

【頁面類型自動判斷】
  有頁首（B=頁首）     → Symbol Page（頁首置頂→大圖→標題→規則）
  無頁首               → Feature Page（標題→規則，自動垂直置中）
  包含賠分表資料       → Paytable Page（頁首→標題→說明規則→賠分格）
  A=轉輪示意           → Reel Diagram Page（左右並排，每頁最多2區塊）

【自動分頁（splitLongSection）】
  基於像素高度計算（非行數），token 寬度以等寬字元替代估算，
  確保長規則頁正確拆分。續頁保留 heroToken（每頁都顯示標題圖片）。

【contentBottom 計算規則】
  有頁尾：min(FTR_BOT - 頁尾高度 - 6, btnTop)
  無頁尾：BTN_Y - BTN_H/2 - 16（≈ 969px）
  → v1.6 起，無頁尾的頁面將原本保留給頁尾的空間釋出給內容區，
    表格與文字可使用更多垂直空間（+32px）。

【垂直置中】
  Feature Page 及含示意圖頁：
  內容高度 < 可用空間 85% 時，自動上移 40% slack，視覺重心略偏上置中。
  Paytable 頁及 Symbol Page（有頁首）不套用此邏輯。

【美化排版（calcRichLayoutParams）】
  純文字/圖示 Feature Page 空間充裕時自動：
  - LINE_H 從 1.8× 拉大至最高 2.6×
  - maxW 從 1700px 收窄至最小 1100px（讓行更短，更有層次感）
  填充目標：頁面使用率 ~80%。
  ※ v1.6 改善：測量函式修正後，含大量 [TOKEN] / {N} 的頁面
    （如 Hold & Spin Feature 說明頁）能正確觸發行距展開。


================================================================================
  Paytable 賠分格排版規則
================================================================================

  上排：最多 5 個符號（WW M1 M2 M3 M4 M5）
  下排：最多 6 個符號（A K Q J 10 9）
  每個符號：圖示（≤150px）在上，賠分列在下，左對齊區塊水平置中
  欄寬：依各符號實際賠分寬度計算（非平均分配），整排水平置中
  間距：兩排之間固定 16px；剩餘空白 35%/57% 分佈於上下
  標籤（如 "OF A KIND"）：顯示在第二排賠分下方


================================================================================
  表格渲染（Pipe Table）
================================================================================

  觸發條件：規則行含 ｜（全形）或 |（半形）
  字體：從 22px 自動縮小至最小 11px，確保所有欄位不溢出
  多行 Cell：超出欄寬的內容自動換行，row 高度動態計算
  表格寬度：依內容自然寬度 × 最多 1.25 倍，不強制滿版，整體置中
  表頭識別：
    - 第 0 行永遠是表頭（金色背景）
    - 後續行若所有欄內容與第 0 行完全相同，也套用金色表頭樣式
  欄對齊：第 0 欄左對齊，其他欄置中

【v1.6 新增：表格行高優化】
  縮放後補足空間（Expansion Pass）：
    若表格行高被 scale-down 壓縮後仍有剩餘空間，
    自動等比放大行高填滿（上限 1.8×），避免表格懸空留白。

【v1.6 新增：同頁多表格統一行高（computePagePipeRowH）】
  同一頁面包含多個管線表格時（如 Symbol 合頁的上下兩表），
  統一計算所有表格的目標行高，確保視覺間距一致，
  不因各表格可用空間不同而出現「上寬下窄」的落差。


================================================================================
  隱式兩欄表格（Implicit 2-Column）
================================================================================

  觸發條件：規則行包含 3 個以上連續空白（全形或半形），且空白兩側都有內容
  欄寬計算：量測所有行（含表頭）的最大內容寬，統一欄寬確保對齊
  分隔線：10個以上短橫線自動渲染為實心金色線，並與兩欄對齊
  注意：表頭行、分隔線、資料行會被收集為同一個 block，共用欄位寬度


================================================================================
  Symbol Page（有頁首）多符號合頁
================================================================================

  當同一群組中相鄰兩個符號可合併在同一頁時，系統自動合頁，
  兩個符號各自以完整 heroSize 尺寸顯示（v1.6 修正：不再縮小為 0.65×）。
  兩個符號之間以半透明金色分隔線區分。
  合頁上限：每頁最多 2 個 section（由 packIntoPages 控制）。


================================================================================
  偵測掉字功能
================================================================================

  按下【🔍 偵測掉字】按鈕：
  1. 對所有頁面在 off-screen canvas 做完整渲染（不影響主畫面）
  2. drawRules / drawRulesReturn 在截斷點（maxY guard）時檢查是否有未顯示行
  3. 有掉字的頁面在左側「頁面列表」標題後方顯示紅字「有掉字」
  4. 狀態列顯示：✅ 全部無掉字 或 ⚠️ 發現 N 頁有掉字
  注意：paytableEntries 的純文字標籤（如 OF A KIND）掉字不在偵測範圍內

  側欄說明
================================================================================

  📁 檔案匯入   Excel / 底圖 / Logo / 圖示資料夾
  💾 匯出       目前頁 PNG | 全部 ZIP
                存 JSON | 載入 JSON
                🔍 偵測掉字
  🔍 素材對應   顯示所有 token 圖示的對應狀態
  📄 頁面列表   點擊跳頁；掉字頁面標紅「有掉字」
  ⚙️ 版面微調
    - 內容區下移安全值（0-180px，預設 60）
    - 大圖與標題距離（0-120px，預設 40）
    - 大圖尺寸（60-420px，預設 180）
    - 大圖 Y 微調（±100px）
    - 規則文字大小（12-32pt，預設 24）
    - 顯示安全框（綠色虛線，標示實際內容區）
    - 顯示頁首/頁尾區域
        黃色：頁首區（HDR_Y ~ SAFE_Y）
        青色：頁尾區（動態計算）
        紅色：按鈕區（不可侵入）
        橘線：內容底線（contentBottom）

================================================================================
  常見問題
================================================================================

  Q: 圖示顯示橘色佔位框？
  A: 請確認圖示資料夾已選取，檔名與 [TOKEN] 名稱一致（不分大小寫）。

  Q: 偵測掉字沒有紅字，但畫面明顯缺字？
  A: paytable 標籤型內容（純文字行）不在偵測範圍，需手動確認。
     其他情況請提供截圖與 JSON 回報。

  Q: JSON 載入後圖示消失？
  A: JSON 只儲存文字與設定。圖片資源（底圖、Logo、圖示資料夾）
     每次開啟後需重新選取。

  Q: 某頁排版跑掉怎麼調？
  A: 使用⚙️版面微調，先試「內容區下移安全值」與「規則文字大小」。
     可開啟「顯示頁首/頁尾區域」確認各區域邊界。

  Q: 同頁兩個表格行距不一樣？
  A: v1.6 已修正，同頁所有管線表格會自動統一行高。
     若仍有異常請使用「🔍 偵測掉字」並提供 JSON 回報。

  Q: Feature Page 行距太緊？
  A: v1.6 改善了含 [TOKEN] / {N} 頁面的高度估算精度。
     若特定頁面仍然過緊，可嘗試降低「規則文字大小」滑桿
     或開啟「緊湊模式」調整後再關閉以重建版面。
